# Верстка оффлайн библиотеки

Программа позволяет сверстать оффлайн библиотеку на основе материалов скачанных утилитой
  ```parce_tululu_category.py``` (https://github.com/learn2410/parse_library2)
  
- пример сайта: https://learn2410.github.io/offline_library   
  
## Запуск скрипта

Для запуска скрипта у вас уже должен быть установлен Python 3.

- Скачайте код
- Установите зависимости командой `pip install -r requirements.txt`
- Запустите скрипт командой:
```
python render_website.py
```  
после генерации html страниц в подпапке "library/pages" программа запустит локальный сайт и откроет первую страницу в браузере, выход из программы по клавишам ```Ctrl+C```

- примечание: для корректной работы программы в подпапке "library" у вас должны находиться скачанные материалы (файл "catalog.json", папка "books" с текстами книг, папка "images" с обложками книг)

## Результат
В результате работы скрипта в папке "library/pages" появятся html странички библиотеки, а в папке со скриптом стартовая страница "index.html". Посмртреть библиотеку в оффлайн можно открыв файл "index.html" в браузере (либо файл "library/pages/index1.html" на который сделана переадресация из "index.html")

## Конфигурация программы
параметры работы скрипта задаются глобальными переменными в начале скрипта:
- LIB_DIR        - подпапка с библиотекой (по умолчанию "library")
в этой подпапке находятся папки указанные в:
- TEXTS_SUBDIR   - папка с текстами книг (по умолчанию "books")
- IMAGES_SUBDIR  - папка с обложками книг (по умолчанию "images")
- PAGES_SUBDIR   - папка куда генерируются html страницы(по умолчанию "pages")
- STATIC_SUBDIR  - папка со статическими файлами (CSS, JS и т.п., по умолчанию "static")
  
примечание: имена папок не должны быть многоуровневыми т.е. не долны содержать 2 и более папок  
 (например "library/xxx/yyy")
 
- BOOKS_PER_PAGE - число книг на создаваемой html странице библиотеки
- PAGINATOR_LEN  - количество страниц отображаемых в пагинаторе (если скачано много книг и это число
 будет большим (например 150) то пагинатор будет отображен некорректно

кроме того в режиме работы локального сайта (когда вы запустили скрипт но еще не нажали ```Ctrl+C```)
можно поэкспериментировать с версткой страниц внося изменения в файл "template.html", при сохранении этого файла изменения должны сразу отобразиться в открытой странице браузера (не забудьте сохранить куда-нибудь оригинальный "template.html")
